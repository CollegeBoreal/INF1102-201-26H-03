## 🎯 Objectif
Mettre en place un environnement Python fonctionnel sous Windows permettant d’utiliser JupyterLab pour le développement et l’analyse de données.

---

## 🧩 Description du projet
Ce travail consiste à installer et configurer Miniforge comme gestionnaire d’environnements Python, puis à utiliser cet environnement pour exécuter JupyterLab.

L’objectif principal est d’obtenir un environnement isolé, stable et adapté au développement, tout en évitant les conflits de dépendances.

---

## ⚙️ Étapes réalisées
- Installation de Miniforge via Chocolatey  
- Configuration de l’environnement PowerShell  
- Ajout des variables nécessaires au système  
- Initialisation de Conda pour permettre la gestion des environnements  
- Création d’un environnement Python dédié  
- Installation de JupyterLab  
- Lancement et vérification du bon fonctionnement  

---

## 🧪 Résultat obtenu
À la fin de l’installation :
- Conda est fonctionnel dans PowerShell  
- Un environnement Python dédié est opérationnel  
- JupyterLab se lance correctement dans le navigateur  
- L’environnement est prêt pour le développement et les projets  

---

## 🧠 Points importants
- L’utilisation d’un environnement virtuel permet d’éviter les conflits entre projets  
- Miniforge offre une alternative légère et efficace à Anaconda  
- La configuration de PowerShell est essentielle pour activer correctement Conda  
- JupyterLab fournit une interface moderne et interactive pour travailler avec Python  

---

## 🧹 Gestion des environnements
Il est possible de créer, supprimer et gérer plusieurs environnements Python selon les besoins des projets, ce qui permet une meilleure organisation du travail.

---

## 📚 Conclusion
Ce projet permet de comprendre comment configurer un environnement de développement Python propre et professionnel sous Windows.  
L’utilisation de Miniforge et JupyterLab constitue une solution efficace, flexible et adaptée aux besoins académiques et professionnels.
